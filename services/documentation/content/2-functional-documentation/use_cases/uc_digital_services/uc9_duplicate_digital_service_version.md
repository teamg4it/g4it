## Description

This use case allows a user to duplicate a digital service version to test different actions and hypotheses in order to understand their consequences on environmental impact. When a user duplicates a version, the resources are copied into a new version.
Users can duplicate the version in two ways: Either they can duplicate the active version directly or they can navigate to manage versions and then duplicate the version.

When user duplicates a digital service version, a new version is created with the same resources as the original version. The new version is set to "draft" status, allowing the user to make modifications without affecting the original version. The name of the duplicated version is automatically generated by appending "(1)" to the original version's name to differentiate it.

**Navigation Path**

-   My Digital Services / Digital Service Version view / Duplicate version
-   My Digital Services / Digital Service Version view / Manage versions / Duplicate version

**Access Conditions**
The connected user must have the write access for that module on the selected workspace.
