## Description

This API allows users to manage eco-design recommendations associated with digital services.
Recommendations are pieces of actionable advice aimed at reducing the environmental impact of a digital service.

Two scopes of recommendations coexist:

- **General recommendations**: not tied to any specific organisation (`organisationId` is null). They are shared across all organisations and are read-only via this API (they cannot be created, updated, or deleted through the organisation-scoped endpoints).
- **Organisation-specific recommendations**: created and managed by a specific organisation, identified by `organisationId`.

All operations are **synchronous** and return responses immediately.

The API call is handled by [RecommendationController](https://github.com/teamg4it/g4it/blob/insa/difficulty/services/backend/src/main/java/com/soprasteria/g4it/backend/apirecommendationds/controller/RecommendationController.java)
and the business logic is handled by [RecommendationService](https://github.com/teamg4it/g4it/blob/insa/difficulty/services/backend/src/main/java/com/soprasteria/g4it/backend/apirecommendationds/business/RecommendationService.java).

## GET – Retrieve recommendations for an organisation

### API Call Processing

{{< mermaid align="center">}}

flowchart LR
A[GET /organizations/{organizationId}/recommendations] --> B(Fetch general recommendations\nwhere organisationId IS NULL)
B --> C(Fetch organisation-specific recommendations\nwhere organisationId = param)
C --> D(Merge both lists)
D --> E(Map entities to RecommendationDSRest DTOs)
E --> F(Return 200 OK with the list)

{{</ mermaid >}}

The response merges **general recommendations** (applicable to all organisations) with recommendations **specific to the requested organisation**. If no recommendations exist, an empty list is returned (HTTP 200).

The retrieval queries are executed by [RecommendationRepository](https://github.com/teamg4it/g4it/blob/insa/difficulty/services/backend/src/main/java/com/soprasteria/g4it/backend/apirecommendationds/repository/RecommendationRepository.java) using two distinct queries:

- `findByOrganisationIdIsNull()` – returns general recommendations.
- `findByOrganisationId(organisationId)` – returns organisation-specific recommendations.

## POST – Create a recommendation

### API Call Processing

{{< mermaid align="center">}}

flowchart LR
A[POST /organizations/{organizationId}/recommendations\nBody: RecommendationDSRest] --> B(Map DTO to Recommendation entity\nidRecommendation ignored auto-generated)
B --> C(Set organisationId on the entity)
C --> D(Persist entity to database)
D --> E(Map saved entity back to RecommendationDSRest DTO)
E --> F(Return 201 Created with the created recommendation)

{{</ mermaid >}}

The `idRecommendation` field is **ignored** on creation: it is auto-generated by the database (identity strategy). The `organisationId` is always taken from the URL path parameter, not from the request body.

## PUT – Update a recommendation

### API Call Processing

{{< mermaid align="center">}}

flowchart LR
A[PUT /organizations/{organizationId}/recommendations/{recommendationId}\nBody: RecommendationDSRest] --> B{Recommendation exists?}
B -- No --> ERR1[Throw 404 – Recommendation not found]
B -- Yes --> C{organisationId matches?}
C -- No --> ERR2[Throw 409 – Recommendation does not belong to this organisation]
C -- Yes --> D(Merge non-null fields from DTO into existing entity\nidRecommendation and organisationId are never overwritten)
D --> E(Persist updated entity)
E --> F(Return 200 OK with the updated recommendation)

{{</ mermaid >}}

The merge strategy uses `NullValuePropertyMappingStrategy.IGNORE`: only non-null fields from the request body are applied to the existing entity. The fields `idRecommendation` and `organisationId` are **never overwritten** during an update.

### Error cases

| HTTP Status | Condition |
|-------------|-----------|
| 404 | The `recommendationId` does not exist in the database |
| 409 | The recommendation exists but belongs to a different organisation than the one in the URL |

## Data Model

### Recommendation entity

The `Recommendation` entity is mapped to the `recommendation` table.

| Package | Entity | Table |
|---------|--------|-------|
| com.soprasteria.g4it.backend.apirecommendationds.modeldb | Recommendation | recommendation |

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id_recommendation` | BIGINT | PK, auto-generated | Unique identifier |
| `title` | VARCHAR | NOT NULL | Short title of the recommendation |
| `description` | TEXT | nullable | Detailed description |
| `category` | JSONB | nullable | List of categories the recommendation belongs to |
| `affected_attributes` | JSONB | nullable | Attributes of the digital service targeted by this recommendation |
| `heuristic_range` | JSONB | nullable | Range of heuristic values associated with this recommendation |
| `base_impact` | INTEGER | nullable | Base environmental impact score |
| `organisation_id` | BIGINT | nullable | Organisation owning this recommendation; NULL for general recommendations |
| `difficulty` | VARCHAR(10) | nullable | Estimated difficulty of applying the recommendation |

### InstantiatedRecommendation entity

The `InstantiatedRecommendation` entity represents the application of a recommendation to a specific digital service version. It is mapped to the `instanciated_recommendation` table.

| Package | Entity | Table |
|---------|--------|-------|
| com.soprasteria.g4it.backend.apirecommendationds.modeldb | InstantiatedRecommendation | instanciated_recommendation |

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id_instantiated_recommendation` | BIGINT | PK, auto-generated | Unique identifier |
| `id_recommendation` | BIGINT | NOT NULL | Foreign key referencing `recommendation.id_recommendation` |
| `digital_service_version_uid` | VARCHAR | NOT NULL | UID of the digital service version this recommendation applies to |
| `priority` | DOUBLE | nullable | Computed priority score for this recommendation in the given context |
| `specific_affected_attributes` | JSONB | nullable | Context-specific override of affected attributes |

> **Note:** The endpoint `GET /recommendations/instantiated` (linked to `InstantiatedRecommendation`) is currently **not yet exposed**: the corresponding controller method and service are commented out pending the implementation of `InstantiatedRecommendationRest`. The repository [InstantiatedRecommendationRepository](https://github.com/teamg4it/g4it/blob/insa/difficulty/services/backend/src/main/java/com/soprasteria/g4it/backend/apirecommendationds/repository/InstantiatedRecommendationRepository.java) is already available and supports lookup and deletion by `digitalServiceVersionUid`.

## Mapper

The mapping between entities and REST DTOs is handled by [RecommendationMapper](https://github.com/teamg4it/g4it/blob/insa/difficulty/services/backend/src/main/java/com/soprasteria/g4it/backend/apirecommendationds/mapper/RecommendationMapper.java), a MapStruct interface with `componentModel = "spring"`.

Key mapping rules:

- On creation (`toEntity`): `idRecommendation` is ignored (auto-generated by the database).
- On update (`merge`): `idRecommendation` and `organisationId` are never overwritten; null values in the source are ignored.
- For `InstantiatedRecommendation`: the `recommendation` nested object and `digitalServiceVersionUid` are explicitly excluded and must be set manually in the service layer.
